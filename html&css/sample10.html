Semantic Elements: describes its meaning to both the browser and the developers

Semantic Elements의 종류 및 설명

  • <article> element specifies independent, self-contained content.
    1. Forum posts
    2. Blog posts
    3. User comments
    4. Product cards
    5. Newspaper articles
  • <aside>: define some content aside from the content it is placed in
  • <details>: Defines additional detail that user can view
  • <figcaption>: caption for a figure element
  • <figure>: specifies self-contained content, like illustartion, diagrams, etc.
  • <footer>: footer for a document or section
    1. authorship info
    2. copyright info
    3. contact info
    4. sitemap
    5. back to top links
    6. related documents
  • <header> element: represents a container for introductory content or a set of navigational links.
    1. one or more heading elements
    2. logo or icon
    3. authorship information
  • <main>: specifies the main content of a doc
  • <mark>: markded text
  • <nav>: set of navigation links.
  • <section>: defines a section in a document.
    1. Chapters
    2. Introduction
    3. New items
    4. Contact information
  • <summary>: visible heading for a details element
  • <time>: date/time

URL: Uniform Resource Locator

scheme://prefix.domain:port/path/filename

  • scheme: defines the type of Internet service (http, https, ftp, file)
  • prefix: defines a domain prefix (default for http: www)
  • domain: defines the internet domain name
  • port: defines the port number at the host
  • path: defines a path at the server
  • filename: the name of a document or resource.
